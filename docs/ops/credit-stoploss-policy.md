# 积分节流与止损策略

## 核心原则

- 直写 GitHub 优先，减少本地落盘与额外补推送
- 长流程交给服务器执行体，减少对话内反复搜索、重复构建和空转等待
- 同一路径、同任务、同阶段失败 2 次即止损，不继续自动重试
- 日志只回传摘要，详细噪音留在服务器状态目录

## 节流手段

- 优先按目标路径、目标模块、目标命令缩小执行范围
- 构建与校验优先使用 `pnpm --filter`，不默认全仓构建
- 多文件改动优先走批量写入，而不是逐文件高频提交
- 任务指纹去重：同一任务未结束前不重复派发新任务
- 若必须先在工作区修改，则由自动跟传助手统一回写 GitHub

## 止损规则

- `maxAttempts = 2`
- 第二次失败后标记为 `blocked`
- `blocked` 状态必须返回：阶段、失败摘要、缺失资源、建议下一步
- 不允许在没有新上下文、新凭据或新条件时继续自动重试

## 人工接管条件

- 缺少真实密钥或仓库写入令牌
- 服务器运行环境与白名单命令不匹配
- 目标仓库权限不足
- 同类错误已达到止损上限

## 摘要格式

- 当前阶段
- 最终结论
- 已写入或未写入的文件范围
- 是否触发止损
- 下一步动作
