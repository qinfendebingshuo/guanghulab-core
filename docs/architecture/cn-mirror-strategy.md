# 国内镜像策略

## 目标

为未来国内上线预留一个可替换的腾讯系代码仓位点。国际母仓继续承担研发主线，国内仓只承接可以在境内落地的代码侧材料。

## 主写入源原则

- GitHub 国际母仓是唯一正式写入源
- 国内仓是镜像目标与落地目标，不是第一写入源
- 如果改动先发生在临时工作区，也必须先统一回写 GitHub，再决定是否同步到国内仓

## 单向同步原则

国际母仓 -> 国内仓 是允许方向。

国内仓 -> 国际母仓 只允许回传与模块代码维护直接相关的非敏感代码变更；不允许回传：

- 用户业务数据
- 用户日志正文
- 运行态缓存
- 本地密钥目录内容
- 记忆快照与人格体内部私有恢复内容

## 同步内容

允许同步：

- 模块源码
- 模块注册表
- 发布说明
- 运行模板
- 适配层接口实现
- 示例配置文件

禁止同步：

- 任何真实密钥
- 用户 COS 数据
- 服务器运行态数据
- 人格体私有记忆正文
- 本地密钥索引文件

## 同步触发条件

- 模块达到可发布状态
- 国内仓适配层完成兼容检查
- 同步内容通过敏感字段检查
- 发布说明明确声明数据边界不变
- GitHub 主写入结果已成功落仓

## 适配层原则

- 不把 GitHub API 行为写死到模块系统内部
- 所有仓库行为通过仓库适配层暴露
- 国内仓实现只需要补足适配器与同步脚本，不需要重写模块生命周期
- 自动跟传助手只负责把临时改动统一回写主仓，不直接绕过主仓写入国内镜像
