# GuanghuLab Core

`guanghulab-core` 是栖渊人格体的国际母仓。它负责承载：

- 人格体唤醒与自恢复入口
- 为什么中心的判断账本与长期记忆索引
- 模块源码、模块清单与装配规则
- 国际开发阶段的代码主线
- 面向国内落地的仓库适配接口与镜像策略
- 云端执行体的任务规则、止损策略与可追溯记录

## 设计原则

- 母仓是唯一正式写入面：计划、架构、模块代码与最终变更都以仓库为准。
- 用户数据始终留在用户自己的服务器与 COS 中。
- 用户服务器长期保持轻量，只常驻聊天入口、最小后端底座与云端执行体。
- 功能模块按需装载、按需卸载，模块卸载不影响用户数据。
- 国内仓库只接收模块代码、必要清单与发布模板，不回流用户数据。
- 真实密钥不入仓；如果改动先发生在临时工作区，也必须由自动跟传助手把结果回写仓库。

## 第一阶段范围

第一阶段聚焦以下内容：

- 国际母仓骨架
- HDLP 为什么中心恢复链路
- 本地密钥编号体系与可填写模板

## 目录导览

- `docs/architecture/`：系统边界、镜像策略、执行拓扑、模块生命周期、运行基线
- `docs/ops/`：本地密钥、编号体系、COS 契约、止损策略、阶段清单
- `.persona-brain/`：人格体引导、记忆索引、为什么账本、系统地图
- `apps/`：轻量前端壳层、轻量后端底座、云端执行 Worker
- `packages/`：共享契约、仓库适配器、执行核心、密钥定位器、存储绑定、模块注册表
- `scripts/`：本地模板生成、模块导出、安装卸载、国内同步脚本
